Basic Everyday Commands

Home

Command
Description
git init
Initializes a new Git repository in the current directory.
git clone [url]
Copies an existing repository (from a URL) to your local machine.
git add [file]
Stages changes made to the specified file(s) for the next commit.
git commit -m "[message]"
Records changes in the repository with a descriptive message.
git status
Displays the current state of the repository, including staged, unstaged, and untracked files.
git log
Shows the commit history of the repository.
git branch
Lists, creates, renames, or deletes branches.
git checkout [branch]
Switches to the specified branch or commit.
git merge [branch]
Combines changes from the specified branch into the current branch.
git pull
Fetches updates from a remote repository and merges them into the local branch.
git diff
Shows the differences between your working directory and the staging area, or between commits.
git rm [file]
Removes a file from the working directory and stages the removal for the next commit.
git stash
Temporarily saves changes that are not ready to be committed and reverts the working directory to a clean state.
git stash pop
Applies the latest stashed changes and removes them from the stash.
git push [remote] [branch]
Uploads commits from the local branch to the specified branch on the remote repository.
git fetch [remote]
Downloads updates from the specified remote repository without merging them into your local branch.
git rebase [branch]
Moves the base of the current branch onto the specified branch, integrating its commits.
git reset [file]
Unstages a file from the staging area while keeping changes in the working directory.
git tag [tagname]
Creates a tag to mark a specific commit (commonly used for releases).
git show [commit/tag]
Displays detailed information about a specific commit or tag.
